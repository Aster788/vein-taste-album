# 产品需求文档 · 旅行美食推荐网站

---

## 一、产品概述

### 1.1 产品定位

一个私人旅行美食推荐网站，以"书"为核心视觉语言，记录用户走过的城市和收藏的美食店。整体气质：手帐风、温暖有质感，让每一位访客感受到幸福感和探索欲。

### 1.2 Slogan

展示格式（中英双语，两行）：

```
世界这本书 我想一直读       ← 沐瑶软笔手写体，较大字号
The world is a book, and I never want to stop reading it   ← Playfair Display，字号缩小，中文下方
```

动画分两阶段执行（详见第三节）：

1. 打字机效果：中文行逐字打出全部10个字，打字完成后英文行淡入显示
2. 打字完成后：两行文字整体呈横幅被风吹动的飘动动画

### 1.3 技术栈


| 层级     | 技术                                                                           |
| ------ | ---------------------------------------------------------------------------- |
| 前端框架   | React（Vite）                                                                  |
| 地图渲染   | Mapbox GL JS（自定义暖白纸感样式：底色米白 `#F7F3EE`，道路浅暖灰，水域淡蓝，高对比度，兼顾纸质感与可读性）             |
| 行政区划边界 | GeoJSON 文件（本地，每个城市一个文件）                                                      |
| 坐标转换   | gcoord 库（GCJ-02 → WGS-84，仅中国城市）                                              |
| 数据     | 本地 JSON 文件（restaurants.json + dishes.json），无后端                               |
| 图表     | Chart.js（雷达图）                                                                |
| 贴纸素材   | SVG， 存放于本地 stickers 文件夹下                                                     |
| 字体     | 霞鹜文楷（本地 `LXGWWenKai-Regular.ttf`）、Playfair Display（Google Fonts）、沐瑶软笔手写体（本地） |
| 部署     | 静态托管（Vercel）                                                                 |


### 1.4 地图数据架构说明

> ⚠️ 重要：高德/谷歌 API 仅用于**数据准备阶段**，网站运行时只调用 Mapbox，不实时请求高德或谷歌。

```
数据准备阶段（建表时手动操作）：
  中国城市店铺   → 用高德 API/App 查询经纬度、地址、营业时间、人均、…… → 填入 Excel
  非中国城市店铺  → 用谷歌 API/Maps 查询经纬度、地址、营业时间、人均、…… → 填入 Excel

网站运行时：
  Mapbox 读取 restaurants.json 中的 lng/lng 字段渲染标注点
  不再实时调用高德或谷歌（避免双SDK冲突）

地图跳转链接：
  中国城市店铺   → map_platform: "amap"，map_url 为高德链接
  非中国城市店铺  → map_platform: "google"，map_url 为谷歌地图链接
```

> ⚠️ 坐标系转换：Mapbox 使用 WGS-84，高德返回 GCJ-02，直接混用导致中国城市标注点偏移约 200–500 米。
> 处理方式：项目引入 `gcoord` 库，读取 restaurants.json 时自动将中国城市经纬度从 GCJ-02 转换为 WGS-84 再传给 Mapbox。
> 非中国城市（济州岛、吉隆坡）使用谷歌坐标，原生 WGS-84，无需转换。

### 2.5 语言切换规则

#### 书脊与封面（11 城统一）

书架与详情页中，**每一本书的书脊与封面**（若展示城市名）采用同一套版式，**不区分中国城与非中国城**：

- **格式**：「国家·城市」——中文行为 `国家中文·城市中文`，英文行为 `Country · City`（英文行中点号两侧保留空格，与中文行的中点 `·` 区分排版习惯）。
- **竖排两行**：**上行为中文**（霞鹜文楷），**下行为英文**（Playfair Display）；两行均为竖向书写（`writing-mode: vertical-rl`）。
- **不随 EN/CN 切换改变**：语言切换只影响页面内其它区域（店铺名、地址、引导语等），**不改变**书脊/封面上的中英双行内容。

> 示例（上行中文 / 下行英文，竖排阅读）：中国城如 `中国·大连` / `China · Dalian`；济州岛如 `韩国·济州岛` / `South Korea · Jeju`；吉隆坡如 `马来西亚·吉隆坡` / `Malaysia · Kuala Lumpur`。

#### 中文城市（中国9城）

- 页面右上角显示 **EN / CN 切换按钮**
- 默认展示中文，点击 EN 切换为英文，点击 CN 切回中文
- 切换影响范围：页面内可变文字（店名、菜系、地址、引导语等），**不含**书脊与封面（见上节「书脊与封面」）

#### 非中文城市（济州岛、吉隆坡）

- 页面右上角**不显示语言切换按钮**
- **详情页等页面主体区域**：店铺名、地址、说明等自动同时展示多语言，固定顺序：**本国语言 → 英文 → 中文**（与书脊/封面的双列中英版式无关）
  - 济州岛（韩语）：韩文 / English / 中文
  - 吉隆坡（马来语）：Bahasa Melayu / English / 中文

## 三、第一页：书架页面

### 3.1 页面结构

- 背景：纯白
- 页面右上角：EN/CN 语言切换按钮（中国城市详情页显示；**首页书架始终显示**）。切换仅影响页面内可变文案，**不影响**书脊与封面（见 §2.5「书脊与封面」）
- 居中显示横向书架，书本**横向排列**，可左右滑动（overflow-x scroll，平滑滚动）
- Slogan 使用沐瑶软笔手写体，书架上方居中展示
- Slogan 旁配贴纸：地球仪（SVG）

### 3.2 Slogan 动画（分两阶段）

**阶段一：打字机效果**

- 页面加载后，中文行「世界这本书 我想一直读」逐字打出
- 每字间隔约 150ms，光标跟随末字闪烁
- 中文打字完成后光标消失，英文行以淡入（fade-in，约 500ms）方式出现
- 英文行：*The world is a book, and I never want to stop reading it*（Playfair Display，字号为中文的60%）

**阶段二：飘动动画**

- 中英两行整体呈横幅被风吹动效果
- 实现方式：CSS keyframes，结合 `translateX` + 轻微 `skewX` + `opacity` 微变
- 动画循环播放，幅度轻柔，不影响文字可读性
- 参考效果：像丝巾在微风中轻轻摆动，而非剧烈晃动

### 3.3 书本设计规范

- 书本为 3D 立体样式，使用 CSS perspective + transform 实现
- 书脊朝外，用户从书脊看这排书
- 书脊文字：竖向两行叠放，**上行中文、下行英文**，格式为「国家·城市」，与 §2.5「书脊与封面」一致，11 城统一（中国城与非中国城同一规则）。示意（上→下）：
  ```
  中国·大连     ← 上行：中文（霞鹜文楷）
  China · Dalian   ← 下行：英文（Playfair）
  ```
  中文行字号略大于英文行，保证窄书脊上可读性。
- 书脊上方：城市代表贴纸（SVG）
- 书本封面（悬停时可见）：城市名与书脊**相同版式**（上中文「国家·城市」、下英文「Country · City」；封面为横向排版时仍保持上/下语义一致）
- 书本颜色：每本书使用对应城市主色，辅以纸张纹理，呈现精装书质感
- 书本厚度、书脊宽度、倾斜角度：对标 Stripe Press 书架视觉（竖版改横版排列）

### 3.4 交互：悬停效果

- 鼠标悬停某本书：该书**向屏幕方向抽出一半**，模拟从书架取书效果
- 动画：平滑过渡（CSS transition，约 300ms，ease-out）
- 其余书本轻微侧移，给抽出的书让出视觉空间

### 3.5 交互：点击跳转

- 点击悬停中（抽出）的书，进入该城市的第二页（路由 `/:city`）
- 翻书过渡动画（**锦上添花，MVP后做**）：书本进一步抽出 → 展开翻成两页 → 自然过渡到第二页

## 四、第二页：城市详情页

### 4.1 页面顶部导航栏

一条横线分为三个色块，全部使用当前城市主色系：

```
[ 城市名色块（左） ] [ 板块①（中，默认选中，按压效果） ] [ 板块②（右） ]
```

- 左色块：显示城市名（中英双语，如「大连 Dalian」），使用城市专属主色背景
- 中色块：数字「1」，默认选中，有向下按压的视觉效果（box-shadow inset）
- 右色块：数字「2」，鼠标移入时呈现按压效果，点击切换到板块②

### 4.2 板块①：地图页

#### 布局比例

```
[ 地图区域（左，约80%宽） ] [ 笔记信息区域（右，约20%宽） ]
```

书本两页摊开，中缝处有淡淡折痕阴影（CSS渐变，宽 8-12px），不影响地图视觉连续性。

#### 地图区域

**基础展示：**

- 使用 **Mapbox GL JS**，基于 Mapbox Light 样式定制：底色改为米白 `#F7F3EE`，道路改为浅暖灰，水域保留淡蓝
- 效果目标：有纸质温暖感，同时保证行政边界、标注点、区名文字高对比度清晰可辨
- 地图展示当前城市范围

**行政区划边界（核心需求）：**

- 通过 GeoJSON 文件在地图上渲染当前城市所有下辖区的边界线
- 每个区的区名以文字标注显示在区域内部（字号适中，清晰可辨）
- 边界线样式：细线，半透明，与地图底色协调，不遮挡店铺标注点
- 目标效果：用户进入页面即可一眼看清城市分区划分，无需任何操作

**店铺标注：**

- 每家收藏美食店，根据经纬度显示标注点（圆点图标，颜色使用城市主色）
- 标注点上方连线引出店铺名称标签（线段连接，标签为圆角小矩形）
- 右侧书脊处：竖向便利贴标签，按一级标签对应的菜系类型分类（如韩餐、日料、中国菜、融合菜）
- 点击菜系标签：对应类型标注点+连线+店铺名称标签高亮，其余变灰；再次点击取消筛选

#### 笔记信息区域（右侧20%）

设计参考：笔记本横线样式，全区域均为横线区

**默认状态（未点击任何店铺）：**
灰色引导文字，沐瑶软笔手写体，居中显示：

> 点击地图上的任意一家店试试看

**点击店铺后：**

```
横线笔记区（全区域）
─────────────────────────────
店铺名称  [地图跳转图标]
─────────────────────────────
详细地址  [文字加超链接的形式，文字前面有地图跳转图标]
─────────────────────────────
菜系类型
─────────────────────────────
综合评分
─────────────────────────────
人均消费
─────────────────────────────
营业时间
─────────────────────────────
联系电话（有才显示，无则不显示）
─────────────────────────────
评分雷达图（4维度，自动切换）
─────────────────────────────

```

- 地图跳转图标：高德或谷歌地图品牌图标（根据 `map_platform` 字段判断），紧跟店名右侧，点击在新标签页打开 `map_url`
- 评分雷达图（Chart.js，搭配数字如 4.2/5）：
  - 堂食（dine_in）：口味 / 环境 / 排队友好度 / 服务 / 总体推荐值
  - 外带（takeout）：口味 / 环境 / 包装 / 排队友好度 / 服务 / 总体推荐值
  - 外卖（delivery）：口味 / 包装 / 配送速度 / 总体推荐值

### 4.3 板块②：杂志/菜单详情页

整体保持打开的书本两页摊开样式，折痕处理与板块①相同。

#### 总体布局

```
左页（40%）：菜系筛选 + 店铺列表
右页（60%）：店铺地址 + 拍立得图片区 + 菜品信息区
```

#### 左页：菜系筛选 + 店铺列表

**菜系筛选（左页顶部）：**

- 视觉呈现：横向标签组，每个一级标签前配置专属食物贴纸 SVG 图标，标签文案与图标一一对应，每个标签后显示对应店铺数量；每个二级标签前无专属食物贴纸 SVG 图标，每个二级标签后显示对应店铺数量
- 标签规则：
一级标签：按国家 / 区域 / 品类划分（如中餐、日料、韩餐、西餐、东南亚菜、饮品、烘焙、其他），列表由后端根据当前收录店铺的菜系数据动态下发，前端无需硬编码；默认标签为“全部”
二级菜单：点击一级标签时，以下拉 / 展开形式展示细分菜系：

场景细分：其他（融合菜、快餐、街头小吃、素食、海鲜等）。
- 交互逻辑：
选中态：点击标签后，标签高亮并触发筛选，下方店铺列表实时刷新为对应菜系的店铺；

## 五、数据结构

### 5.1 主数据文件：restaurants.json

所有城市所有店铺在一个 JSON 数组里，按 `city_en` 字段筛选。

```json
[
  {
    "city_zh": "大连",
    "city_en": "Dalian",
    "country_zh": "中国",
    "country_en": "China",
    "is_china": true,

    "name_zh": "店铺中文名",
    "name_en": "",
    "cuisine": "韩餐",
    "address": "大连市中山区XX路XX号",
    "lat": 38.9140,
    "lng": 121.6147,
    "price_per_person": 80,
    "currency": "CNY",
    "hours": "10:00–22:00",
    "phone": "",
    "map_platform": "amap",
    "map_url": "https://...",

    "dining_type": "dine_in",

    "score_taste": 4.5,
    "score_service": 4.0,
    "score_queue": 3.5,
    "score_overall": 4.5,
    "score_packaging": null,
    "score_delivery": null
  }
]
```

字段说明：

- `is_china`：布尔值，用于判断是否需要 GCJ-02→WGS-84 坐标转换，以及是否显示语言切换按钮
- `currency`：货币单位，中国城市填 `CNY`，济州岛填 `KRW`，吉隆坡填 `MYR`
- `name_en`：可为空字符串，中文餐厅无需强制填写
- 堂食时 `score_packaging` 和 `score_delivery` 填 `null`；外带时 `score_delivery`填 `null`；外卖时 `score_environment`, `score_service` 和 `score_queue` 填 `null`

### 5.2 菜品数据文件：dishes.json

独立文件，与 restaurants.json 通过 `store_name_zh`（中国城市）或 `store_name_local`（非中国城市）关联。

```json
[
  {
    "store_name_zh": "店铺中文名",
    "store_name_local": "",
    "city_en": "Dalian",

    "name_zh": "红烧带鱼",
    "name_en": "Braised Cutlassfish",
    "name_local": "",
    "price": "38",
    "taste": 5
  }
]
```

> **变更说明**：`dishes.json` **不包含 `review` 字段**；Excel **dishes 表不设 `review` 列**。菜品长文字评价不在本文件中维护；板块②「评价层」**仅展示口味星级**（由 `taste` 渲染，见 4.3 节）。

字段说明：

- `name_local`：非中国城市菜品的本国语言名称（如韩文、马来文），中国城市填空字符串
- `name_zh` / `name_en`：所有城市都填，便于多语言展示
- `**price`：一律为 JSON 字符串（`string`）**。纯金额写作 `"38"`；按件/重量等计价写作 `"19/个"`、`"24/对"`、`"10/斤"` 等，前端原样展示，不解析为数字。
- `taste`：JSON **数字**（`number`），主观口味档位，取值 **整数 `1`、`2`、`3`、`4`、`5`**，分别对应 **一星～五星** 的展示强度。**数据层只存数字，不存 `⭐` 等字符**；页面上由**统一工具函数**将数值渲染为星标（如 Unicode `⭐` 重复 `taste` 次，或等价空心/实心方案，全站一套规则）。若日后需半档（如 `4.5`），再扩展同一渲染函数与建表约定，不在 JSON 手写星串。板块②展示见 4.3 节。
- 图片文件名即为菜名（中国城市用 `name_zh`，非中国城市用 `name_local`），代码通过文件名匹配 dishes.json 中对应条目
