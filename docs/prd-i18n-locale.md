## 一、核心规则（来源：prd.md §2.5）

### 2.5 语言切换规则

#### 书脊与封面（12 城统一）

书架与详情页中，**每一本书的书脊与封面**（若展示城市名）采用同一套版式，**不区分中国城与非中国城**：

- **格式**：「国家·城市」——中文行为 `国家中文·城市中文`，英文行为 `Country · City`（英文行中点号两侧保留空格，与中文行的中点 `·` 区分排版习惯）。
- **竖排两行**：**上行为中文**（霞鹜文楷），**下行为英文**（Playfair Display）；两行均为竖向书写（`writing-mode: vertical-rl`）。
- **不随语言切换改变**：语言切换只影响页面内其它区域（地址、引导语、字段标签等），**不改变**书脊/封面上的中英双行内容。

> 示例（上行中文 / 下行英文，竖排阅读）：中国城如 `中国·大连` / `China · Dalian`、`中国·苏州` / `China · Suzhou`；济州岛如 `韩国·济州岛` / `South Korea · Jeju`；吉隆坡如 `马来西亚·吉隆坡` / `Malaysia · Kuala Lumpur`；马六甲如 `马来西亚·马六甲` / `Malaysia · Melaka`。

#### 中文城市（中国9城）

- 中文城市**详情页**右上角显示 **EN / CN 切换按钮**（书架页不显示）
- 默认展示中文，点击 EN 切换为英文，点击 CN 切回中文
- 切换影响范围：页面内可变文字（店名、菜系、地址、引导语等），**不含**书脊与封面（见上节「书脊与封面」）

#### 非中文城市（济州岛、吉隆坡、马六甲；以及未来新增的日本、泰国等）

- **详情页**右上角显示语言切换按钮（书架页不显示）；按钮集合按城市配置：
  - **默认（大多数非中国城市）**：`EN / <ISO639-1> / CN`（例如济州岛：`EN / KO / CN`）
  - **英语通用度极高的国家/地区（当前口径：马来西亚·吉隆坡）**：简化为 `EN / CN`（不展示第三语言按钮）
- 默认展示 **CN（中文）**；点击 **EN** 切换为英文；点击 **本国语按钮**（如 `KO`）切换为对应语言；点击 **CN** 切回中文
- 切换影响范围：页面内可变文字（**城市名**、标签名、菜系、引导语、字段标签如「综合评分/人均/营业时间」等），**不含**书脊与封面（见上节「书脊与封面」）

##### 名称类字段（不参与语言切换、不做机器翻译）

以下字段始终以作者在 `restaurants.json` / `dishes.json`（由 Excel 同步）中提供的内容为准：

- **店铺名**：`restaurants.json` 的 `name_zh` / `name_en` / `name_local`
- **菜品名**：`dishes.json` 的 `dish_name_zh` / `dish_name_en` / `dish_name_local`

##### 其它可变文案（允许机器翻译补齐 + 回退链）

对「名称类字段」以外的可变文案（如地址、说明、引导语、字段标签等），按当前语言取值规则：

1. **优先使用作者提供的原文**（若 Excel/JSON 已为该语言准备独立列/字段，则直接读取）
2. **若该语言缺失**：允许调用机器翻译服务补齐（需可关闭；需记录缓存，避免重复扣费）
3. **若机器翻译失败/未启用**：按回退链 **A：`目标语言 → 英文 → 中文`** 展示可用文本

##### Map 页：地图点位「店铺名标签」展示规则（固定优先级）

地图点位上方引出的店铺名标签属于**店铺名展示**，**不跟随**右上角语言切换；固定取值为：

- `name_zh` 有值 → 用 `name_zh`
- 否则 `name_en` 有值 → 用 `name_en`
- 否则 `name_local` 有值 → 用 `name_local`
- 三者皆空 → 不展示标签（或展示占位符「未命名店铺」，产品后续再定；MVP 可先不展示）

## 二、与页面模块的联动规则（从原 §4.2 / §4.3 抽离）

### 2.1 Map 页（板块①）

- 地图点位店铺名标签属于名称类字段展示，不跟随详情页右上角语言切换。
- 固定优先级：`name_zh -> name_en -> name_local`；三者均空可不展示。
- 右侧笔记区中字段标签（如综合评分、人均、营业时间）随当前语言策略切换。

### 2.2 杂志详情页（板块②）

- 店铺列表排序与语言切换解耦：切换 `EN/CN/本地语` 仅影响展示文案，不触发重排。
- 非中国城市店名展示固定多行顺序：`name_zh -> name_en -> name_local`，不参与语言切换。
- 菜系文案展示与语言切换解耦：切换语言不改变已确定的筛选顺序与逻辑。
- 图片侧菜名遵循名称类字段规则：优先展示作者提供的多语言名称，不做机器翻译覆盖。

## 三、统一验收清单（供各 Phase 引用）

- 详情页语言按钮集合正确：
  - 中国城市：`EN/CN`
  - 非中国城市：`EN/<ISO639-1>/CN` 或 `EN/CN`（按 `city_meta.json`）
- 书架页不显示语言切换按钮。
- 书脊与封面文案不随切换变化。
- 名称类字段（店名/菜名）不随切换变化、不走机器翻译。
- 其它可变文案遵循「作者原文优先 -> MT 补齐 -> A 回退链」。
- Map 标签与板块②店名展示符合“解耦 + 固定优先级/顺序”规则。
