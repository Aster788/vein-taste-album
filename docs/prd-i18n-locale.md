## 一、核心规则（来源：prd.md §2.5）

### 2.5 语言切换规则

#### 书脊与封面（13 城统一）

书架与详情页中，**每一本书的书脊与封面**（若展示城市名）采用同一套版式，**不区分中国城与非中国城**：

- **格式**：「国家·城市」——中文行为 `国家中文·城市中文`，英文行为 `Country · City`（英文行中点号两侧保留空格，与中文行的中点 `·` 区分排版习惯）。
- **竖排两行**：**上行为中文**（霞鹜文楷），**下行为英文**（Playfair Display）；两行均为竖向书写（`writing-mode: vertical-rl`）。
- **不随语言切换改变**：语言切换只影响页面内其它区域，**不改变**书脊/封面上的中英双行内容。

> 示例（上行中文 / 下行英文，竖排阅读）：中国城如 `中国·大连` / `China · Dalian`、`中国·南京` / `China · Nanjing`、`中国·苏州` / `China · Suzhou`；济州岛如 `韩国·济州岛` / `South Korea · Jeju`；吉隆坡如 `马来西亚·吉隆坡` / `Malaysia · Kuala Lumpur`；马六甲如 `马来西亚·马六甲` / `Malaysia · Melaka`。

#### 中文城市（中国10城）

- 中文城市**详情页**右上角显示 **EN / CN 切换按钮**（书架页不显示）
- 默认展示中文，点击 EN 切换为英文，点击 CN 切回中文
- 切换影响范围：页面 UI 文案（Tab、字段标签、引导语、空状态等）、菜系**筛选展示名**（下拉/地图便利贴按钮文案）、顶栏城市标题；**basename 精确匹配**的板块②菜名（`dish_name_zh` ↔ `dish_name_en`，见下节「菜品名」）。**不含**书脊与封面、店铺列表/地图/笔记区店名、地址正文、笔记区 `cuisine_zh` 数据值。

#### 非中文城市（济州岛、吉隆坡、马六甲；以及未来新增的日本、泰国等）

- **详情页**右上角显示语言切换按钮（书架页不显示）；按钮集合按城市配置：
  - **默认（大多数非中国城市）**：`EN / <ISO639-1> / CN`（例如济州岛：`EN / KO / CN`）
  - **英语通用度极高的国家/地区（当前口径：马来西亚·吉隆坡）**：简化为 `EN / CN`（不展示第三语言按钮）
- 默认展示 **CN（中文）**；点击 **EN** 切换为英文；点击 **本国语按钮**（如 `KO`）切换为对应语言；点击 **CN** 切回中文
- 切换影响范围：页面 UI 文案（**城市名**、标签名、菜系筛选展示名、引导语、字段标签如「综合评分/人均/营业时间」等）。**不含**书脊与封面、店铺名固定多行展示、地址正文、笔记区 `cuisine_zh` 数据值。板块②精确匹配菜名固定多行 `dish_name_zh` → `dish_name_en` → `dish_name_local`（不随按钮缩成单行）。

##### 店铺名（不参与语言切换、不做机器翻译）

以下始终以作者在 `restaurants.json`（由 Excel 同步）中提供的内容为准，**不**随 EN/CN/本地语按钮切换：

- **店铺名**：`name_zh` / `name_en` / `name_local`
- 板块②列表：按 `getCuisineDisplayNameLines` 固定多行展示（中国城通常首行 `name_zh`）
- 地图点位标签、笔记区店名：固定优先级规则，见下节 Map 规则

##### 菜品名（参与语言切换；不做机器翻译）

板块②图片侧菜名（`DishInfo`）：

- **basename 与 `dishes.json` 菜名精确一致**时视为「命中该菜品」：
  - **中国城**：随 `detailLocale` 在 `dish_name_zh` ↔ `dish_name_en` 间切换（`pickByLocale`，**`allowMachineTranslate: false`**）；缺英文时回退中文。价格、星级、备注仍读该条 `dishes.json`。
  - **非中国城**：固定多行 `dish_name_zh` → `dish_name_en` → `dish_name_local`（有值则展示，**不**随按钮隐藏某语言行）。
- **basename 仅包含菜名但非完全一致**（如 `贵州抹茶柚子蛋糕-横截面.jpg`）：**不**走菜名切换，仅显示完整 basename，不挂价格/星级/备注（见 [project_rules.md](project_rules.md) §五）。
- 实现见 [DishInfo.jsx](../src/components/DishInfo.jsx)。

##### 其它可变文案（允许机器翻译补齐 + 回退链）

对「店铺名」以外的可变文案（如地址、说明、引导语、字段标签等；**不含**已单独规定的菜品名、菜系筛选展示名），按当前语言取值规则：

1. **优先使用作者提供的原文**（若 Excel/JSON 已为该语言准备独立列/字段，则直接读取）
2. **若该语言缺失**：允许调用机器翻译服务补齐（需可关闭；需记录缓存，避免重复扣费）
3. **若机器翻译失败/未启用**：按回退链 **A：`目标语言 → 英文 → 中文`** 展示可用文本

##### Map 页：地图点位「店铺名标签」展示规则（固定优先级）

地图点位上方引出的店铺名标签属于**店铺名展示**，**不跟随**右上角语言切换。先按字段取完整店名，再按是否多分店决定是否去括号（见 `docs/prd.md` §5.1「地图标签店名」、`pickMapTagDisplayName`）：

- `name_zh` 有值 → 用 `name_zh`
- 否则 `name_en` 有值 → 用 `name_en`
- 否则 `name_local` 有值 → 用 `name_local`
- 三者皆空 → 不展示标签（或展示占位符「未命名店铺」，产品后续再定；MVP 可先不展示）
- **单店**（同城 `store_slug` 仅一条 `branch`）：标签去掉末尾 `()` / `（）` 及括号内内容
- **多分店**（同城同 `store_slug` 的 `branch` ≥ 2）：标签保留完整店名（含括号分店后缀）
- 点击标签后，右侧笔记区店名仍展示完整 `name_zh`，不受标签去括号影响

## 二、与页面模块的联动规则（从原 §4.2 / §4.3 抽离）

### 2.1 Map 页（板块①）

- 地图点位店铺名标签属于店铺名展示，不跟随详情页右上角语言切换。
- 固定优先级：`name_zh -> name_en -> name_local`；单店标签去括号、多分店保留完整名（见 §5.1）；三者均空可不展示。
- 地图右侧店铺详情店名展示完整 `name_zh`，与标签文案规则分离。
- 右侧笔记区中字段标签（如综合评分、人均、营业时间）随当前语言策略切换；笔记区菜系**值**固定 `cuisine_zh`。

### 2.2 杂志详情页（板块②）

- 店铺列表排序与语言切换解耦：切换 `EN/CN/本地语` 不触发重排。
- 店铺列表店名：固定多行规则，**不参与**语言切换（中国城与非中国城均如此）。
- 菜系**筛选 UI**（下拉/按钮展示文案）随语言切换；筛选键仍为 `cuisine_en`；排序逻辑不变。详见 [structure.md](structure.md) §菜系筛选贴纸。
- 菜品名：见上节「菜品名（参与语言切换）」；精确匹配走多语言字段，前缀变体图仅 basename。

## 三、统一验收清单（供各 Phase 引用）

- 详情页语言按钮集合正确：
  - 中国城市：`EN/CN`
  - 非中国城市：`EN/<ISO639-1>/CN` 或 `EN/CN`（按 `city_meta.json`）
- 书架页不显示语言切换按钮。
- 城市详情页**始终**显示「返回书架」按钮（`BookshelfBackButton` → `/`），与语言切换显隐无关；视觉见 `prd-ui-spec.md` §4.1。
- 书脊与封面文案不随切换变化。
- **店铺名**不随切换变化、不走机器翻译。
- **中国城精确匹配菜名**随 EN/CN 在 `dish_name_zh` / `dish_name_en` 间切换（无 MT）；**非中国城精确匹配**固定三语多行。
- 前缀变体图（如 `菜名-横截面`）仅显示 basename，不挂菜品信息。
- 其它可变文案遵循「作者原文优先 -> MT 补齐 -> A 回退链」。
- Map 标签与板块②店名展示符合“解耦 + 固定优先级/顺序”规则。
