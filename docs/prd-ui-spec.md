### 2.3 全局视觉语言

- **背景**：纯白 `#FFFFFF`，整个网站底色统一
- **纸质感**：书本、笔记区域使用纸张纹理（CSS noise texture 或 SVG 纹理叠加）
- **书本质感**：精装书、厚重感，书脊有明显厚度的 3D 透视
- **书本折痕**：两页摊开时，中缝用淡淡渐变阴影（宽 8–12px，CSS box-shadow），不用实线；折痕位置跟随内容布局边界动态定位（优先对齐地图区与笔记文字区临界线），不固定在页面 50%

## 三、第一页：书架页面

### 3.1 页面结构

- 背景：纯白
- 页面右上角：书架页**不显示**语言切换按钮；该按钮仅在**城市详情页**显示（中国城：`EN/CN`；非中国城：按 §2.5）。切换仅影响页面内可变文案，**不影响**书脊与封面（见 §2.5「书脊与封面」）
- 居中显示横向书架，书本**横向排列**，可左右滑动（overflow-x scroll，平滑滚动）
- Slogan 使用沐瑶软笔手写体，书架上方居中展示
- Slogan 旁配贴纸：地球仪（SVG）

### 3.2 Slogan 动画（分两阶段）

**阶段一：打字机效果**

- 页面加载后，中文行「世界这本书 我想一直读」逐字打出
- 每字间隔约 150ms，光标跟随末字闪烁
- 中文打字完成后光标消失，英文行以淡入（fade-in，约 500ms）方式出现
- 英文行：*I wander, I wonder, I fall in love with the world-again and again*（沐瑶软笔手写体，字号为中文的60%）

**阶段二：飘动动画**

- 中英两行整体呈横幅被风吹动效果
- 实现方式：CSS keyframes，结合 `translateX` + 轻微 `skewX` + `opacity` 微变
- 动画循环播放，幅度轻柔，不影响文字可读性
- 参考效果：像丝巾在微风中轻轻摆动，而非剧烈晃动

### 3.3 书本设计规范

- 书本为 3D 立体样式，使用 CSS perspective + transform 实现
- 书脊朝外，用户从书脊看这排书
- 书脊文字：竖向两行叠放，**上行中文、下行英文**，格式为「国家·城市」，与 §2.5「书脊与封面」一致，13 城统一（中国城与非中国城同一规则）。示意（上→下）：
  ```
  中国·大连     ← 上行：中文（霞鹜文楷）
  China · Dalian   ← 下行：英文（Playfair）
  ```
  中文行字号略大于英文行，保证窄书脊上可读性。
- 书脊上方：城市代表贴纸（SVG）
- 书本封面（悬停时可见）：城市名与书脊**相同版式**（上中文「国家·城市」、下英文「Country · City」；封面为横向排版时仍保持上/下语义一致）
- 封面阶段性视觉规则（当前实现口径）：封面先使用**纯色**，且颜色与对应书脊主色一致；纹理/插画后续再补，不影响交互开发与验收
- 书本颜色：每本书使用对应城市主色，辅以纸张纹理，呈现精装书质感
- 书脊字色：默认按 `--city-primary` 亮度自动选深/浅字；**苏州、马六甲**主色饱和度较高，强制使用浅色书脊字（`Book.jsx` 中 `FORCE_LIGHT_SPINE_INK_SLUGS`），保证窄书脊可读性
- 书本厚度、书脊宽度、倾斜角度：对标 Stripe Press 书架视觉（竖版改横版排列）

### 3.4 交互：指针分界 + 命中抽出（Stripe Press 式）

- **指针竖线分界**：以光标水平位置 `mx` 为分界；书中心在光标**左侧**的书，随与光标水平距离减小逐渐露出**正面（封面）**；书中心在光标**右侧**的书，随与光标水平距离减小逐渐露出**反面**（当前 3D 为书脊+书口+封面，反面以书口/脊背侧为主），且**反面最大转角小于正面**。
- **横向滚动**：仅用于浏览书排；书转角不依赖「视口中心 / 滚动位置」余弦规则。
- **抽出（translateZ）**：仅当光标**落在该书槽矩形内**（非「靠近但未触书」）时触发；过渡 `300ms ease-out`。
- **立体**：仍以书脊为轴的 `rotateY` 为主，配合透视与现有材质高光/阴影。

### 3.5 交互：点击跳转

- 点击该书链接进入该城市的第二页（路由 `/:city`）
- 脚印导航（页面底部）：一本书对应一个脚印；点击任一脚印后，书架横向平滑滚动并将对应书本定位到视口中间
- 书本定位完成后，对应脚印保持深灰高亮，其余脚印维持浅灰
- 翻书过渡动画（**锦上添花，MVP后做**）：书本进一步抽出 → 展开翻成两页 → 自然过渡到第二页

### 4.2 板块①：地图页

#### 布局比例

```
[ 地图区域（左，约70%宽） ] [ 笔记信息区域（右，约20%宽） ] [ 便签筛选区域（右，约10%宽） ]
```

书本两页摊开，中缝处有淡淡折痕阴影（CSS渐变，宽 8-12px）。折痕不固定在 50% 位置，而是对齐地图区域与笔记信息区域的临界线；当地图区与笔记区占比调整时，折痕位置需同步动态调整。

#### 地图区域

**基础展示：**

- 使用 **Mapbox GL JS**，基于 Mapbox Light 样式定制：底色改为米白 `#F7F3EE`，道路改为浅暖灰，水域保留淡蓝
- 效果目标：有纸质温暖感，同时保证行政边界、标注点、区名文字高对比度清晰可辨
- 地图展示当前城市范围
- 地图加载状态与异常提示（必须）：
  - 正常加载时：地图正常渲染，不显示错误提示
  - `VITE_MAPBOX_TOKEN` 为空时：显示「缺少 `VITE_MAPBOX_TOKEN`，地图无法初始化」
  - Token / 网络 / 样式请求失败时：显示「地图加载失败，请检查 Mapbox Token 或网络连接」并附带错误细节；不得长期停留在「地图加载中...」

**行政区划边界（核心需求）：**

- 通过 GeoJSON 文件在地图上渲染当前城市所有下辖区的边界线
- 每个区的区名以文字标注显示在区域内部（字号适中，清晰可辨）
- 边界线样式：细线，半透明，与地图底色协调，不遮挡店铺标注点
- 目标效果：用户进入页面即可一眼看清城市分区划分，无需任何操作

**店铺标注：**

- 数据源：`getMappableRestaurantsByCity`（`branch` + 有效坐标，且排除 `brand`、`closed=yes`、`address=连锁店`；菜品页仍展示被排除的店）
- 每家**地图可见**的收藏美食店，根据经纬度显示标注点（圆点图标，颜色使用城市主色）
- 标注点上方连线引出店铺名称标签（线段连接，标签为圆角小矩形）
- **店铺名标签文案**：属于「店铺名展示」，**不跟随**详情页右上角语言切换；固定优先级为 `name_zh → name_en → name_local`（字段定义见 §5.1；三者皆空则不展示标签）。实现：`pickMapTagDisplayName`（[`src/utils/storeGroups.js`](../src/utils/storeGroups.js)）
- **单店标签**：同城该 `store_slug` 仅一条 `branch` 时，标签去掉末尾括号及括号内分店后缀（例：`3号仓库·创意中国菜(五角场店)` → `3号仓库·创意中国菜`）
- **多分店地图标签**：同城同 `store_slug` 的 `branch` 行 ≥ 2 时，各 branch 仍各打一个点、各一条标签；标签 React key 与 GeoJSON `store_key` 按 `(city_en, store_slug, lng, lat)` 唯一，禁止仅用 slug。每条标签保留**完整**店名（例：`南里山房(上海来福士广场店)`）。空间不足时由 `formatMapTagLabel` 从标签文案末尾截断加省略号，**禁止**单独展示括号内分店后缀（如仅显示 `打浦桥店` / `荟聚店`）
- **点击标签后的店铺详情**：笔记区店名展示完整 `name_zh`（不受标签去括号影响）
- 右侧书脊处：竖向便利贴标签，按一级标签对应的菜系类型分类（如韩餐、日料、中国菜、融合菜）
- 点击菜系标签：对应类型标注点+连线+店铺名称标签高亮，其余变灰；再次点击取消筛选

#### 笔记信息区域（右侧约20%）

设计参考：笔记本横线样式，全区域均为横线区

**默认状态（未点击任何店铺）：**
灰色引导文字，沐瑶软笔手写体，居中显示：

> 点击地图上的任意一家店试试看

**点击店铺后：**

─────────────────────────────
详细地址  [文字加超链接的形式，文字前面有地图跳转图标]
─────────────────────────────

─────────────────────────────
综合评分（有才显示，无则不显示）
─────────────────────────────
人均消费（有才显示，无则不显示）
─────────────────────────────
营业时间（有才显示，无则不显示）
─────────────────────────────
联系电话（有才显示，无则不显示；`restaurants.xlsx` 中多个号码以英文 `;` 或中文 `；` 分隔时，展示层逐行换行，不显示分隔符）

- 评分雷达图（Chart.js，搭配数字如 4.2/5）：
  - 维度字段来源：`score_taste` / `socre_environment` / `score_service` / `score_queue` / `score_packaging` / `score_delivery` / `score_personal`
  - 中文映射：口味 / 环境 / 服务 / 排队友好度 / 包装 / 配送 / 个人推荐值
  - 渲染规则：不再按 `dining_type` 写死维度集合；以上字段中**有值就显示该维度**，为空或 `null` 就隐藏该维度
  - 交互规则：雷达图不显示 tooltip（悬停不弹出提示文案）

#### 便签筛选区域（右侧约10%）

- 位置：贴靠书脊一侧，独立于笔记正文区
- 内容：竖向便利贴菜系标签（与地图点位筛选联动）
- 交互：点击后对应类型高亮，其余变灰；再次点击取消筛选
- 排版与宽度规则（强约束）：
  - 按钮文案采用“首行左对齐、后续行居中”规则；若仅一行则按首行规则左对齐
  - 分行策略为“先判定是否超宽，再决定是否拆行”；未超宽保持单行，超宽才拆行
  - 按钮宽度随文案自适应；按钮内左右留白需保持一致（代码与视觉一致）
  - 宽度上限按文案脚本判定（与页面语言切换解耦）：
    - 中文 `4em`
    - 英文 `12em`
    - 日文 `4.5em`
    - 韩文 `5.5em`
    - 泰文 `6.2em`
  - 英文文案统一小写展示（不使用首字母大写）
- 滚动规则（强约束）：
  - 标签数量未超出可视区时不显示滚动条
  - 标签数量超出可视区时显示纵向滚动条
  - 滚动条颜色需与当前城市主色保持一致（含悬浮态）
- 菜系数据与贴纸：`restaurants.json` 使用 `cuisine_zh`（中文展示，含地图右侧标签文案）+ `cuisine_en`（贴纸 slug，**与** `src/assets/stickers/cuisine/{cuisine_en}.svg` **文件名一致**，如 `chuan-cuisine.svg`）。`cuisine_en` 为贴纸与筛选键的准绳；缺 SVG 时回退 `other.svg`。见 [docs/structure.md](structure.md) §菜系筛选贴纸、`src/utils/cuisineSlugs.js`

### 4.3 板块②：杂志/菜单详情页

整体保持打开的书本两页摊开样式，折痕处理与板块①相同。

#### 总体布局

```
左页（40%）：菜系筛选 + 店铺列表
右页（60%）：店铺地址 + 拍立得图片区 + 菜品信息区
```

#### 左页：菜系筛选 + 店铺列表

**菜系筛选（左页顶部）：**

- 视觉呈现：`下拉按钮 + 下拉框列表`（非横向标签组），下拉框每项为 `贴纸（由 cuisine_en 加载）+ 菜系名（cuisine_zh）+ 店铺总数`（不显示序号）
- 位置：放在顶部 `切换按钮 + Cuisine` 这一行下方，与其左侧基线对齐
- 标签规则：默认项为“全部”，且“全部”始终位于第一个
- 交互逻辑：
选中态：选择菜系后触发筛选，下方店铺列表实时刷新为对应菜系店铺；
重置：选择“全部”时恢复全量店铺。
- 下拉列表排序：
  - 首项固定“全部”
  - 其余按店铺数量 `count` 降序
  - `count` 相同按菜系中文拼音 `A-Z` 排序
  - 数字开头条目统一排在字母后
  - 排序结果与语言切换解耦：切换 `EN/CN/本地语` 仅影响显示文案，不改变列表顺序

**店铺列表（音乐播放列表样式）：**

布局结构：

- 垂直单列，所有列表项沿中轴线左对齐
- 单列表项结构：`序号 + 店铺名称`，序号与名称固定间距
- 列表区宽度固定，高度自适应，与右页按 3:7 比例划分
- **列表去重（多分店，通用）**：按 `(city_en, store_slug)` 合并；同城同 slug 的任意多条 `branch` 在列表中只占一项（地图仍按每条 branch 打点）。实现：[`src/utils/storeGroups.js`](../src/utils/storeGroups.js) — **禁止**在 React 中按店名维护白名单
- **店铺名称展示规则**：
  - 店铺名不参与语言切换；展示**基础店名**（不含括号分店后缀）及 `name_en` / `name_local` 非空行，顺序：`name_zh` → `name_en` → `name_local`（每个语言单独一行）
  - 基础名优先级：该组 `dishes.json` 中首条菜品的 `store_name_*` → 否则各分店 `name_zh` 去括号后的最短非空结果
  - 第二行、第三行（若存在）相对首行缩进两个字符
- 店铺列表排序：
  - 按展示用 `name_zh`（基础名）中文拼音 `A-Z` 排序
  - 基础名为空时回退 `name_en`，再回退 `name_local`
  - 数字开头条目统一排在字母后
  - 排序前统一 `trim` 并去除前导符号（标点/特殊符号）
  - 同 key 时保持原始输入顺序（稳定排序）
  - 排序结果与语言切换解耦：切换 `EN/CN/本地语` 不触发重排

滚动与数量：

- 店铺数量未超出可视区时不显示滚动条
- 店铺数量超出可视区时显示纵向滚动条，支持上下滑动
- 列表最后一行固定显示 `AND MORE TO COME~`，作为默认尾行并占用一个序号位（非交互项）

默认状态：

- 页面加载时，默认选中第一个列表项，右侧自动加载对应店铺内容
- 未选中项：无高亮，背景与页面一致
- 鼠标悬浮：文字轻微高亮或行高微变，提示可点击

选中状态：

- 整行使用城市主色高亮
- 同一时刻只有一个选中项；点击新项时，旧项高亮自动取消

**地址展示（方案二）：**

- 列表项仅保留「序号 + 店铺名称」，不在列表内展示地址
- 地址展示位置：右页最上方（见右页说明）

#### 右页：店铺详情

**右页顶部：地址区**

- 位置：右页最上方，左对齐
- 样式：左侧地点类小图标 + 右侧店铺详细地址，小字展示；多行地址使用 `white-space: pre-line` 换行
- 与下方图片区间隔 3 行距离
- 静态展示，随店铺切换自动更新
- **单分店组**（仅一条 branch）：单行 `address`，无前缀
- **多分店组**：每行 `{分店标签}：{address}`；`分店标签` 默认从 `name_zh` 末尾括号内解析（支持 `()` / `（）`），解析失败则用完整 `name_zh`；各行按分店标签拼音排序
- 异常处理：组内所有分店均无地址时，此区域自动隐藏，不占位

**右页主体：拍立得图片区**

图片展示（左60%）：

- 拍立得样式：白边框、轻微随机旋转角度（±3°以内）、轻阴影
- 照片左上角/右上角有回形针或图钉贴纸装饰（SVG）
- 默认加载该店铺第一张图片
- 支持图片放大/缩小（点击放大，再点击或按 ESC 关闭）

图片切换：

- 图片下方页码点（● ○ ○ ○），可点击跳转
- 图片左右各有上一张/下一张按钮
- **轮播顺序（四段，实现见 `src/utils/storePhotos.js`）**：
  1. basename 匹配 `dishes.json` 菜名（`dish_name_local` → `dish_name_en` → `dish_name_zh`）
  2. basename 匹配店名（`dishes.json` 的 `store_name_*`、去括号基础名、同城同 slug 分店 `restaurants.json` 的 `name_*`）
  3. 其余非中文数字序号：英文起头按英文字母序，中文起头按拼音序（数字起头介于英文与中文之间）
  4. 中文数字序号（`一二三四五六七八九十` 及 `十一/二十` 等）按数值升序

菜品信息区（右40%，与图片并列）：

- 位置：图片右侧，与图片顶部对齐
- 内容分两层，从上到下排列：

```
【菜名层】
从图片文件名（去掉扩展名后的 basename）匹配 `dishes.json`，再按字段展示菜名
中国城市：按语言切换展示（中文/英文）
非中国城市：`dish_name_zh` / `dish_name_en` / `dish_name_local` 只要有值就全部显示，每行一个（固定顺序，避免不同条目顺序乱跳）
若 basename 未匹配到任何菜品：
  - basename 为中文数字序号（`一二三四五六七八九十`）时：仅展示图片，不显示菜名文本
  - 其它 basename：显示 basename（不含扩展名）作为图片名称（常用于风景/环境图）
常见呈现：
  - 中国城市：通常显示中文菜名（`dish_name_zh`）
  - 非中国城市：按固定顺序输出非空行：`dish_name_zh` → `dish_name_en` → `dish_name_local`
  - 命中菜品且 `price` 与 `currency` 有值时：在菜名首行后追加价格，格式为「菜名 + 两个空格 + 币种符号+price」
    - 示例：`锅包肉  ¥38`、`흑돼지  ₩18000`、`辣炒年糕  ¥19/个`
  - 非中国城市多行菜名时，仅首行追加价格，后续语言行不重复追加
  - 若 `price` 已包含币种文本/符号则按原值显示；否则按 `currency`（ISO 4217）动态解析并补币种符号（如 `JPY->¥`、`THB->฿`）；若运行环境无法解析则回退为 `currency` 代码前缀

【评价层】（菜名下方）
- **口味星级**：读取 `dishes.json` 中该菜品条目的 **`taste` 数值（整数 1～5）**，由前端**统一函数**渲染为星标（`1` → 一颗星 … `5` → 五颗星）；数据仅存数字，界面不直接展示原始数字作为口味主视觉。
- 本层**不包含**菜品长文字评价：`dishes.json` **不设 `review` 字段**（见 5.2 节）。
- 备注文字：读取 `dishes.json` 的 `note`；有值才显示，无值不显示。若当前图片未匹配到菜品条目，则不显示备注。
- 星级随当前展示图片切换同步更新。
```

- 字体：霞鹜文楷（中文）/ Playfair Display（英文）
- 字号：菜名略大，星级区略小于菜名
